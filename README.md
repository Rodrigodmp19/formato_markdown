# Mardown: Historia, utilidad y casos de uso

## 1) ¿Qué es Markdown?
- Es un lenguaje de marcado que permite dar formato a texto plano de manera sencilla y legible. Se escribe usando caracteres comunes como #, *, -, <, >, etc y esto hace que se pueda convertir en HTML o algun otro formato.

## 2) Origen e historia de Markdown
- Fué creado por John Gruber con la colaboración de Aaron Swartz en el año 2004. Con el propósito de que los documentos fueran fćiles tanto para leerlos y escribirlos en texto plano para poder convertirlos en lenguaje HTML válido.

## 3) ¿Para qué sirve Markdown?
- Para generar HTML, PDF, DOCX, presentaciones, etc.
- Crear contenido sin usar editores visuales.
- Escribir documentos estructurados rápidamente

## 4) Casos de uso reales
- Desarrollo y tecnología: Archivos README.md (como éste). Documentacion técnica. Comentarios en issues y pull requests.
- Escritura y contenido: Blogs. Apuntes académicos. Notas personales.
- Educación y ciencia: Informes. Articulos científicos. Material educativo.
- Productividad: Documentación interna de empresas. Manuales simples. Bases de conocimiento.

## 5) Ventajas y desventajas

## Ventajas:
- Fácil de aprender.
- Muy legible como texto plano.
- No depende de software específico.
- Compatible con muchas plataformas.
- Ideal para control de versiones (Git).

## Desventajas:
- Limitado para diseños complejos.
- No estandarizado al 100% (hay variantes).
- No es ideal para documentos muy visuales.
- Requiere conversión para formatos finales (PDF, HTML).
